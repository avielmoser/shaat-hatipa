# PDF Export QA Checklist

This document outlines the manual verification steps for the improved PDF export functionality (RTL support, Translations, Clinic Branding).

## Prerequisites
1.  Run the app locally (`npm run dev`).
2.  Navigate to the schedule view (complete the wizard).

## Test Cases

### 1. Default Branding (English)
- **Setup**: Use the default clinic (no clinic param) or `?clinic=default`. Ensure language is English.
- **Action**: Click "Export to PDF".
- **Expectation**:
    - [ ] Document is **LTR**.
    - [ ] Title includes "ShaatHaTipa".
    - [ ] Logo is present (ShaatHaTipa logo).
    - [ ] Columns are: "Time", "Drops", "Notes".
    - [ ] "Generated by ShaatHaTipa" in footer.

### 2. Default Branding (Hebrew)
- **Setup**: Default clinic. Switch language to Hebrew.
- **Action**: Click "Export to PDF".
- **Expectation**:
    - [ ] Document is **RTL** (Logos/Text aligned correctly).
    - [ ] Columns are: "שעה", "טיפות", "הערות" (Right-to-Left order).
    - [ ] "נוצר ע״י שעת הטיפה" in footer.

### 3. Custom Clinic Branding (Ein Tal)
- **Setup**: Navigate with `?clinic=ein-tal`.
- **Action**: Click "Export to PDF".
- **Expectation**:
    - [ ] **Top Logo** is the "Ein Tal" logo.
    - [ ] Title includes "Ein Tal".
    - [ ] Action label matches clinic config (if specific label exists) or falls back to "Drops"/"טיפות".

### 4. Layout Mechanics
- **Action**: Print Preview.
- **Expectation**:
    - [ ] Tables do not overflow horizontally.
    - [ ] Chips (Medications) have legible colors on white background.
    - [ ] Text is not cut off.
